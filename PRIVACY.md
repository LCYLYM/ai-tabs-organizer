# AI Tabs Organizer 隐私说明

本项目是一个本地运行的 Chrome 扩展，用于根据用户自定义分类对标签页进行自动分组。为实现该功能，扩展会处理与当前标签页相关的最小必要数据。

## 处理的数据

- 用户主动配置的数据
  - 自定义分类名称
  - 补充提示词
  - Provider 类型
  - OpenAI 兼容接口的 `baseUrl`、`apiKey`、`model`
  - 分组颜色、折叠规则、界面语言与自动扫描参数
- 浏览器标签页数据
  - 当前标签页 URL
  - 页面标题
  - 域名
  - 页面描述、标题层级与正文节选
  - 标签页分组状态与分组结果
- 诊断与历史数据
  - 最近 Provider 状态
  - 最近运行日志
  - 分类结果历史

## 数据如何使用

- 以上数据仅用于完成扩展的单一目的：为用户的标签页做 AI 分类、分组、历史回看与诊断。
- 分类判断遵循固定优先级：`页面标题 > 域名 > 页面正文内容`。
- 当页面正文不可读取时，扩展会退回到标题和域名进行判断。
- 自动打标只会在标签页未获得焦点、页面已加载完成、且当前页面尚未处于分组状态时触发。

## 数据存储位置

- 扩展配置、分类历史、诊断状态存储在本地 `chrome.storage.local`。
- 扩展会调用 `chrome.storage.local.setAccessLevel({ accessLevel: 'TRUSTED_CONTEXTS' })`，限制这些数据仅在受信上下文中可访问。
- 本项目不会把这些数据同步到开发者自建服务器。

## 第三方数据传输

### 1. OpenAI 兼容接口模式

- 当用户选择 `OpenAI 兼容接口` 模式时，扩展会把完成分类所需的页面信号发送到用户自己配置的 `baseUrl`。
- 发送内容通常包括：
  - 页面标题
  - 域名
  - 页面 URL
  - 页面语言
  - 页面描述
  - 标题层级摘要
  - 正文节选
  - 用户自定义分类与补充提示词
- `apiKey` 仅用于向用户配置的接口发起请求，保存在本地扩展存储中，不会上传到本项目作者的服务器。
- 第三方接口服务商如何处理这些数据，取决于用户自行选择的服务提供方及其隐私政策。

### 2. Chrome 内置 AI 模式

- 当用户选择 `Chrome 内置 AI` 模式时，分类尽量在本地 Chrome 内置模型中完成，不需要把页面内容发送到外部 AI API。
- 该模式的可用性与模型下载行为由 Chrome 与设备环境决定。

## 数据共享

- 除用户明确配置的第三方 AI 接口外，本项目不会主动把用户数据共享、出售或转让给广告平台、数据经纪商或其他无关第三方。
- 本项目不会把数据用于广告投放、信用评估或与单一功能无关的用途。

## 用户控制能力

- 用户可以随时修改或清空：
  - 当前窗口分组与打标记录
  - 全部标签页分组与打标记录
  - 分类结果历史
  - Provider 配置与自动打标开关
- 用户卸载扩展后，扩展本地存储的数据将随扩展一并移除。

## 安全说明

- 网络请求使用浏览器标准 `fetch`，预期通过 HTTPS 与用户配置的 API 服务通信。
- 由于扩展需要在用户访问的网页上读取标题、域名和可访问正文内容，Manifest 中声明了广泛的 `http/https` host permissions。这些权限仅用于实现标签页自动分类功能。
- 扩展不执行远程托管的扩展脚本代码；OpenAI 兼容模式下仅向用户指定接口发送标准 API 请求。
